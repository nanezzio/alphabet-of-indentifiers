# Алфавит идентификаторов

Тестовое задание на разработку библиотечного класса генерации идентификаторов.

## Описание

Проект реализует библиотечный класс для генерации следующего идентификатора в последовательности по заданным правилам.

Каждый идентификатор состоит из одной или нескольких групп формата:

```text
<буква><цифра>
```

Группы разделяются символом `-`.

Примеры идентификаторов:

```text
A1
A2
A9
B1
Z9
A1-A1
A1-A2
A2-A1
```

---

## Правила формирования идентификаторов

### Допустимые буквы

```text
A B C E H I K L N O P R S T U W X Y Z
```

Следующие буквы не используются:

```text
D F G J M Q V
```

### Допустимые цифры

```text
1 2 3 4 5 6 7 8 9
```

Цифра `0` не используется.

### Примеры последовательности

```text
A1
A2
A3
...
A9
B1
...
Z9
A1-A1
A1-A2
...
A1-Z9
A2-A1
...
```

После последнего значения группы (`Z9`) создаётся новая группа слева:

```text
Z9 -> A1-A1
```

Перенос между группами работает аналогично переносу разрядов в позиционной системе счисления:

```text
A1-Z9 -> A2-A1
```

Максимальная длина идентификатора — 10 групп.

```text

```

## Алгоритм

Каждая группа идентификатора рассматривается как один разряд системы счисления.

Количество допустимых состояний одной группы:

```text
19 букв × 9 цифр = 171 состояние
```

Таким образом идентификатор представляет собой число в системе счисления с основанием 171.

Генерация следующего идентификатора выполняется как операция увеличения числа на единицу с переносом между группами.

---

## Сложность алгоритма

Пусть N — количество групп в идентификаторе.

Временная сложность:

```text
O(N)
```

Пространственная сложность:

```text
O(N)
```

Максимальное значение N ограничено 10.
